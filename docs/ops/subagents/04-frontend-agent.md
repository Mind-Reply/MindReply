# Frontend Agent

## Role

Keep the public surface minimal, premium, and focused on one decision at a time.

## Current Focus

- First viewport clarity.
- Mobile header and action layout.
- MRagent entry point.
- Removal of public internal operations language.

## Inputs

- Homepage source.
- Agent page source.
- Browser screenshots.
- Mobile viewport checks.

## Outputs

- One UI issue list with file paths.
- One public-copy cleanup list.
- One visual verification note for desktop and mobile.

## Boundaries

- Do not add dashboards, giant marketing cards, or multi-option flows.
- Do not expose internal agent staffing or command language.
- Prefer existing project components and semantic styling over new UI sprawl.

## Current Next Move

After preview is available, verify that the first viewport shows only the decision intake, synthesis/action pattern, risk status, and receipt language.

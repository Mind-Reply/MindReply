# MindReply Website Audit Action Blueprint

This is the current website clarity, trust, conversion, sales-readiness, and deployment blueprint for all MindReply agent lanes.

## Revenue-First Rule

The Website Completion Package is the first explicit paid offer. Pause work that does not improve conversion, activation, close rate, repeat paid usage, deployment continuity, or owner-controlled assisted close.

## Two-Layer Homepage Model

Layer 1: immediate operational relief.

- Lead with `Reclaim 2+ hours daily within 24 hours`.
- Show that MindReply turns emails, Slack notes, tasks, follow-ups, and website confusion into one action queue or send-ready message.
- Keep the first value visible before checkout through MRagent.

Layer 2: premium authority.

- Explain discipline-specific language, behavioral expression refinement, measurable communication structure, privacy-safe receipts, and risk/confidence labels.
- Avoid abstract platform theatre. Authority must make the buyer trust the package, not decode a vague ecosystem.

## Paid Entry Offer

Use `Website Completion Package` as the neutral delivery bundle.

Required framing:

- overloaded website messaging
- ranked action queue
- send-ready copy or reply structure
- privacy-safe receipt
- `GBP 600`
- invoice or direct payment route
- MRagent first, assisted close second

Do not ask visitors to buy technology. Ask them to buy clarity, queue resolution, communication rescue, website completion, reply recovery, and message refinement under pressure.

## Trust Proof

Every public page must keep proof restrained and evidence-based.

Required public proof:

- public contact email is `info@mind-reply.com`
- no public personal Gmail address
- no fake active staff claims
- no invented revenue, subscriptions, credits, billing, or Pro access
- receipts are privacy-safe and do not expose raw private pressure
- Slack, email, memory, MCP, and security lanes are only described as active when credentials, consent, and workflow evidence exist

## SEO And Demand Lanes

Position the site around high-friction, underserved buyer intent:

- website buying-friction rescue
- Website Completion Package
- response overload
- client follow-up pressure
- founder communication rescue
- assisted close
- ranked action queue
- send-ready copy
- privacy-safe receipt
- multilingual business communication support

The sitemap and robots contract must allow the money pages: `/website-completion-package`, `/pricing`, `/contact`, `/capabilities`, `/agent`, and `/`.

## Mobile And Visual Quality

Phone views must not feel like an afterthought.

- No overlapping text or buttons.
- Stable dimensions for chat, package cards, footer blocks, and language selector.
- Keep the language selector gentle and useful, not dominant.
- Keep footer substantial enough to signal trust, demand lanes, evidence rule, and next step.

## Deployment Gate

Do not call production green until live proof confirms:

- homepage contains `Reclaim 2+ hours daily within 24 hours`
- `/contact` contains `info@mind-reply.com` and no `gmail.com`
- `/api/version` returns status `ok`
- `/api/health` returns status `ok`
- public pages include `Website Completion Package` and `GBP 600`
- `/api/package-request` rejects invalid input with HTTP `400`

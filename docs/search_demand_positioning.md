# MindReply Search Demand Positioning

Date: 2026-06-09

## Purpose

This is the internal SEO operating brief for the homepage and owner reports. It keeps the site focused on buyer intent instead of broad AI claims.

## Priority Lanes

1. AI decision support
2. AI email reply assistant
3. Website conversion rescue
4. AI search visibility
5. Premium communication assistant

## Revenue Rule

Lead with the Website Completion Package. The public page should make the fixed GBP 600 package, invoice route, MRagent trial, and buyer-ready website rescue obvious before expanding integrations.

## Claim Boundary

Use only claims that are visible in the product or verified by configuration:

- MRagent can produce a synthesis, next action, and risk-aware reply structure.
- Website Completion Package is the fixed-scope paid offer.
- Language selection supports English, Spanish, French, German, Italian, Portuguese, Dutch, Polish, Ukrainian, and Bulgarian.
- Country detection reads deployment/request headers only and does not store IP addresses.
- Email, Slack, memory, payment, or deployment claims must not be described as active unless credentials and delivery proof exist.

## Automation Hook

The backend endpoint `/api/seo` exposes the current positioning manifest. Future scheduled reports should check that:

- homepage metadata still uses the premium reply intelligence positioning;
- the Website Completion Package appears in the first viewport;
- package CTAs still include GBP 600 context;
- mobile and desktop keep a language option;
- public copy avoids unverified integration claims.

# Security Policy

MindReply is built around privacy-first defaults and owner-controlled decisions.

## Boundaries

- Mail intake starts with header and snippet only.
- Sensitive input is classified locally when possible.
- Raw content is excluded from audit receipts by default.
- High-risk communication requires verification before execution.
- Owner export requires verified owner identity and explicit consent.
- Prepared owner mail contains synthesis, recommended action, redaction level, and receipt identifier.
- Signed receipts contain metadata, synthesis, action, rationale, and redaction level.

## Reporting

Send security reports to `info@mind-reply.com` with the subject `MindReply Security`.
